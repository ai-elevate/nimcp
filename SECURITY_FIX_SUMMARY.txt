SUMMARY OF CHANGES
--------------------------------------------------------------------------------

BEFORE (VULNERABLE):
- Used rand/srand for cryptographic key generation
- Only ~32 bits of entropy (from time, PID, clock)
- Keys were predictable and could be brute-forced
- Violated CWE-338, NIST SP 800-90A, FIPS 140-2

AFTER (SECURE):
- Primary: libsodium randombytes_buf() (when NIMCP_ENABLE_ENCRYPTION defined)
- Fallback Linux/Unix: /dev/urandom (kernel CSPRNG)
- Fallback BSD/macOS: arc4random_buf()
- Fallback Windows: BCryptGenRandom (CNG API, FIPS 140-2 compliant)
- Full 256 bits of cryptographic entropy
- Unpredictable, suitable for long-term cryptographic use

IMPLEMENTATION DETAILS
--------------------------------------------------------------------------------

Primary Implementation (libsodium):
    randombytes_buf(key, NIMCP_SECURITY_KEY_SIZE);

Fallback Unix/Linux:
    1. Open /dev/urandom with O_RDONLY
    2. Read 32 bytes with proper partial read handling
    3. Retry on EINTR (signal interruption)
    4. Fall back to arc4random_buf() on BSD if /dev/urandom unavailable
    5. Close file descriptor

Fallback Windows:
    BCryptGenRandom(NULL, key, 32, BCRYPT_USE_SYSTEM_PREFERRED_RNG);

Thread Safety:
