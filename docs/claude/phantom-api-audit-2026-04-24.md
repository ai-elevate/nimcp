# Phantom API Audit — 2026-04-24

**Scope**: `include/cognitive/` + `include/core/brain/` + `include/security/` (1,424 headers).
**Methodology**: Python-based scanner (`docs/claude/phantom_audit_script.sh`) that
(a) extracts every `... name(...);` prototype from headers, (b) extracts every
top-level function definition from `src/**/*.c` and `include/**/*.h`, (c) expands
common code-generation macros (`CONNECT_IMPL`, `BRIDGE_DEFINE_BIO_ASYNC_FUNCS*`,
`BRIDGE_DEFINE_SECURITY_SETTERS*`, `BRIDGE_DEFINE_MESH_REGISTRATION`,
`HOOK_SETTER`, and local `*_SETTER` macros with `prefix##_set_##name` patterns),
and (d) diffs the two lists. Symbols present in `build/lib/libnimcp.so` via
`nm -D` were spot-checked and confirm the macro-expanded definitions.

### Top-line finding

> **No phantom has a live caller after macro expansion.** The codebase builds
> cleanly because every "declared-but-not-in-src/" function either (a) lives
> in the header's own `#ifdef`-guarded in-header implementation block (pattern
> used extensively by `*_quantum_bridge.h` files), or (b) is generated by a
> `BRIDGE_DEFINE_*` / `HOOK_SETTER` / `CONNECT_IMPL` / local `*_SETTER` macro.
> Everything else is aspirational code with zero call sites.

## 7. Reproducing this audit

```bash
bash /home/bbrelin/nimcp/docs/claude/phantom_audit_script.sh 2>/dev/null \
  | awk -F'\t' '$3 >= 1'                          # CRITICAL candidates (all FP today)
bash /home/bbrelin/nimcp/docs/claude/phantom_audit_script.sh 2>/dev/null \
  | awk -F'\t' '$3 == 0 && $2 ~ /\/cognitive\//'  # MEDIUM cognitive-surface phantoms
```

Known limitations of the scanner:

1. Pseudo-functions inside struct-field comments (`γ_AB (J/m²)`) can be
   misparsed; the strict `name(...) ;` requirement eliminates most, but 5 FPs
   remain (all libc / struct-access / Doxygen — see §3).
2. Some macro code-gen patterns not yet modeled: `FINANCIAL_*_SETTERS`, a
   handful of site-local `DEFINE_*` macros in `cognitive/fault_tolerance/`.
   These contribute at most ~50 additional false positives in the MEDIUM tier.
3. In-header implementations (Header-Only Library pattern, used by all
   `*_quantum_bridge.h` files in `include/cognitive/curiosity,emotion,memory,
   executive,reasoning/`) ARE recognized as implementations, so those 60+
   quantum-bridge symbols are correctly NOT flagged.
